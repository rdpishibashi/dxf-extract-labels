# DXF Extract Labels - 機器符号抽出

DXFファイルからテキストラベルを抽出し、Excel形式で出力するStreamlitアプリケーションです。

## 機能

- **複数ファイル対応**: 複数のDXFファイルを一括処理
- **ラベル抽出**: DXFファイルからTEXT、MTEXTエンティティを抽出
- **機器符号（候補）抽出**: 図面枠内・図面情報欄外のラベルからReference Designator
  パターンに一致し除外パターンに該当しないものを対象にする。確実に機器符号と判定
  できる形（確定パターン）は自動採用され、それ以外は「未確定ラベル」として一覧
  表示し、採用するものを選択できる（チェックしたものだけが出力される）
- **図面番号抽出**: 図番と流用元図番を自動識別（ラベルベース・座標ベース）
- **タイトル・サブタイトル抽出**: 図面のタイトルとサブタイトルを自動抽出
- **複数図面対応**: 1ファイルに複数図面がある場合、全図面枠を対象
- **矩形領域抽出**: 図面枠内の矩形（直交ポリゴン）領域を検出し、領域内ラベルに領域名を付与（「領域選択オプション」）
- **端子一覧抽出**: タイトルが「UNIT内結線図」の図面を対象に、TBで始まる端子台ラベルに
  対応する矩形（LINE+CIRCLEで構成）を検出し、矩形内の端子番号を「TB List」シートに出力
- **Excel出力**: 統計情報とラベル一覧をExcel形式で出力
- **自動集計**: ファイルごとの個別シートとサマリーシートを生成
- **判断ログの記録**: 「未確定ラベル」での採用/非採用の判断を記録・蓄積し、確定/除外
  パターンの見直しに活用（後述）

## 使用方法

### 基本的なワークフロー

1. **ファイルのアップロード**: 複数のDXFファイルをアップロード可能
   - ドラッグ&ドロップまたはファイル選択
   - 複数ファイルの同時アップロード対応

2. **オプション設定** (任意):
   - **機器符号（候補）以外も抽出**: OFF（既定）では図面枠内・図面情報欄外の
     ラベルのうち機器符号パターンに一致し除外パターンに該当しないものが対象。
     確実に機器符号と判定できる形（確定パターン）は自動採用され、それ以外は
     「未確定ラベル」として一覧表示し、採用するものを選択する。
     ON では図面枠制限・フィルタなしで全ラベルを抽出する（従来動作）
     - 候補パターン例: `CNCNT`, `FB`（英字のみ）、`R10`, `CN3`, `AAC1B4-07`（英字+数字+ハイフン任意）、
       `CN-IF2-1`（英字-英字+数字）、`FB()`, `MSS(MOTOR)`, `R10(2.2K)`（括弧付き、括弧より前で判定）
   - **図面番号抽出**: 図番と流用元図番を自動識別
   - **端子一覧を抽出**: タイトルが「UNIT内結線図」の図面（サブタイトルが「TB COMPONENT」
     の図面を除く）を対象に、TBで始まる端子台ラベルに対応する矩形を検出し、矩形内の
     端子番号を「TB List」シート（Totalシート直後）に出力する
   - **並び替え**: 昇順、降順、並び替えなしを選択

3. **ファイル処理**: 「ラベルを抽出」ボタンをクリックして抽出開始

4. **未確定ラベルの選択** (機器符号（候補）以外も抽出＝OFF時のみ):
   - ファイルごとに、機器符号パターンに一致し除外パターンに該当しなかったラベルの
     うち、確定パターンにも一致しなかったもの（一覧・個数、10件ごとに1テーブル）が
     固定幅のテーブルで横並びに表示される（確定パターンに一致したラベルはここに
     表示されず、自動的に採用される）。ブラウザーの横幅に応じて横に並ぶテーブルの
     数が自動調整される（狭ければ2列、広ければ3列、さらに広ければ4列以上）
   - Reference Designator として採用するものにチェックを入れる（初期状態は全てOFF）
   - 末尾の数字（1〜2桁）を除いた部分が同じラベル（例: `CN1`・`CN2`・`CN10`）と
     同一ラベルは、チェックした瞬間に全ファイル横断で連動して採用/解除されます
     （テーブルをまたいでいても即座に反映されます）
   - 「選択完了」をクリックすると、チェックしたラベルと確定分がExcelに出力される
   - このとき、レビュー対象だった全ラベルの採用/非採用が判断ログとして記録されます
     （下記「判断ログの記録」参照。記録に失敗しても抽出処理自体は継続します）

5. **結果のダウンロード**: Excelファイルをダウンロード
   - ダウンロード後は「新しい抽出を開始」ボタンが青色になります
   - 「新しい抽出を開始」をクリックすると、アップロード済みファイルがクリアされます
     （オプション設定は保持されます）

### 出力ファイル

デフォルトのExcelファイル名: `extracted_labels.xlsx`

Excelファイルの構成（既定モード）:
- **Summaryシート**: 全ファイルの抽出結果概要
  - ファイル名、図面枠内ラベル数、機器符号（候補）数、未確定ラベル数（未採用）
  - 図番、流用元図番（図面番号抽出が有効な場合）
  - タイトル、サブタイトル（抽出された場合）
- **各ファイルシート**: 個別ファイルの機器符号一覧（確定パターン自動採用分＋未確定ラベルUIで採用したもの）

「機器符号（候補）以外も抽出」ON 時は、フィルタなしの全ラベルが同様の構成で出力されます。

### 矩形領域抽出（領域選択オプション）

図面枠（太さ100の線で囲まれた枠）内の矩形領域を検出し、領域内ラベルに領域名を付与します。
オプションの「領域を検出」にチェックを入れると使えます（既定OFF）。

1. オプションの「領域を検出」にチェック
2. （任意）「領域検出の詳細設定」で検出パラメータを調整し「設定完了」
   - 「抽出したい領域名の先頭文字列（カンマ区切り・複数可）」を指定すると、名称
     候補がその文字列で始まる領域は面積比の閾値を満たさなくても抽出対象になります
     （前方一致。全角/半角は区別しません）
3. 「領域を検出」をクリック
4. 「領域の確認」で各領域の名称候補をチェックボックスで選択（各領域の📐角座標も確認可能）
   - 「図面番号・タイトル・サブタイトルを抽出」が有効な場合、ファイル名の直下に
     「図番：… / タイトル：… / サブタイトル：…」が表示されます
   - 候補群の末尾の「（領域選択しない）」にチェックを入れると、この領域にはどの
     ラベルも割り当てられません
5. 「ラベルを抽出」→「Excelをダウンロード」（`領域` 列・`領域一覧` シート付き）
   - `領域一覧` シートの `ページ No.` はファイル内の図面枠の通し番号です（DXF図番ではありません）
   - `領域別ラベル一覧` シートでは、検出した領域名ごとに全ファイル横断でラベルと個数を確認できます
     （同名の領域は複数ファイルにまたがって合算されます）

### 端子一覧抽出

タイトルが「UNIT内結線図」の図面（サブタイトルが「TB COMPONENT」の図面＝端子台の
全端子番号一覧ページを除く）を対象に、TBで始まりその直後に英大文字・数字が1文字
以上続くラベル（例: `TB001`、`TBN241 (24A)`。`TB取付板`のようにTBの直後が漢字等の
ものは対象外）に対応する矩形（LINE+CIRCLEで構成、CIRCLEによる橋渡しが1箇所以上
ある矩形のみ）を検出し、矩形内の端子番号（数字のみのラベル）を抽出します。

1. オプションの「端子一覧を抽出」にチェック
2. 「ラベルを抽出」→「Excelをダウンロード」
   - `TB List` シート（`端子台` / `端子No.` / `図番`）が `Total` シート直後
     （領域モードでは `領域別ラベル一覧` 直後）に追加されます
   - **「端子台」でユニーク**をとります。同じ端子台名が複数ファイルにまたがって
     登場する場合、端子番号・図番を1行に統合します（例:
     `TBN241 (24A) | 1, 5, 6, 29, 30, 31 | EE6492-039-38A, EE6892-039-03A`）
   - 「端子No.」で同じ番号が複数回登場する場合は `7(2)` のように件数を
     カッコ内に表示します
   - 「端子台」はABC順、複数の「図番」もABC順にソートされます
   - 候補パターンには一致したが対応する矩形が見つからないラベルは、
     一覧の末尾に1行空けたうえで「端子台」欄に「端子検出不可」、
     「端子No.」欄にそのラベル、「図番」欄にそのファイルの図番として
     記載されます

### 判断ログの記録

「未確定ラベル」で「選択完了」を押すと、レビュー対象だったラベル（採用したもの・
しなかったもの両方）が判断ログとして記録されます。蓄積したログは
`tools/reference_designator_analyzer.py`（開発者向けツール）で分析し、確定パターン・
除外パターンの候補提案に使えます。記録先は実行環境によって自動的に切り替わります。

| 実行環境 | 記録先 | 設定方法 |
|---------|--------|---------|
| Streamlit Community Cloud | GitHub のログ専用リポジトリ | アプリの Secrets に `[github]` セクション（`token`/`repo`）を設定 |
| ローカル PC・Windows アプリ | `~/Documents/DXF-extract-labels/decision_log.csv` | 既定のまま。保存先を変えたい場合は環境変数 `DXF_DECISION_LOG_PATH` を設定（Dropbox 等の共有フォルダを指定すれば複数 PC のログを一元化できます）|

Streamlit Cloud での Secrets 設定例（アプリ管理画面の「Secrets」に貼り付け）:

```toml
[github]
token = "ghp_..."                          # ログ専用リポジトリへの書き込み権限を持つトークン
repo = "your-org/your-log-repo"            # アプリ本体とは別のリポジトリを指定
```

Secrets が未設定の環境（ローカル等）では自動的にローカルファイルへの記録に切り替わり、
エラーにはなりません。GitHub への記録に失敗した場合も抽出処理は止まらず、記録できな
かった分を CSV としてその場でダウンロードできます。
